# FontAgit 30일 그로스 플랜 — 트래픽 0 탈출

> 작성일: 2026-07-16 | 성격: 마스터플랜 v3.0(2차 개정)의 실행 문서 (전략은 마스터플랜, 전투 계획은 이 문서)
> 전제 제약: 마케팅 예산 0원, 창업자 익명 활동만 가능, 마케팅 가용 시간 주 5시간
> 성공 기준: **공개(런칭) 후 30일, GA4 신규 사용자(new users) 누적 1,000명** — 지표는 GA4 신규 사용자로 고정(세션-페이지뷰 아님). 로드맵 유지 결정에 따라 "작성일 기준 30일"에서 "공개 후 30일"로 재설정

## 0. 결론

- 로드맵(S0~S4)은 유지한다. 대신 지금부터 30일의 목표를 "런칭 준비 완료 + 시딩 자산 완비"로 정의하고, 방문 1,000명은 공개 시점부터 30일로 잰다.
- SEO(검색엔진 최적화)는 신규 도메인 특성상 공개 후 60~90일까지 유입을 만들지 못한다. **첫 1,000명은 전부 비SEO 채널에서 온다.** 이 문서의 존재 이유다.
- 개발은 AI(Claude/Codex 세션)가, 배포(시딩)는 창업자가 맡는 2트랙. 창업자의 주 5시간은 전액 시딩 자산 구축과 배포 활동에 쓴다.
- 킬 스위치: 공개 후 14일 내 누적 방문 250명 미만 그리고 제작사/커뮤니티 공유 2건 미만이면 포지셔닝-채널을 재검토한다.

## 1. 적대적 리뷰 종합 (2026-07-16, Opus 심층 + Codex 독립 교차)

두 리뷰가 독립적으로 합의한 치명 결함 5개와 이 문서의 대응:

| # | 결함 | 대응 |
|---|---|---|
| 1 | 신규 도메인 30일 색인 불가 — 로드맵 S3 "2~4주 색인률 50%"는 비현실(실제 60~90일) | 30일 성공 기준을 색인이 아닌 비SEO 직접 유입으로 전환(본 문서) |
| 2 | RPM(1,000뷰당 광고 수익) 가정 낙관 — 한국 폰트 니치 실측은 ₩300~600 가능성. 월 30만 원에 월 60만 PV 필요할 수 있음 | AdSense는 30일 목표에서 제외. 승인 후 실측으로 마스터플랜 2장 보정 |
| 3 | "주 3~5시간 운영으로 해자 축적"은 수학 붕괴 — 주간 루틴 실공수는 주 10~15시간 | 운영 루틴 중 자동화 가능분(수집-링크 점검-이미지)은 AI/파이프라인으로 이관, 사람 시간은 검수와 배포에만 |
| 4 | 초기 Top 10은 "가짜 트렌드" — 클릭 데이터 없는 랭킹은 신뢰를 깎음 | 정직성 게이트(마스터플랜 3-1 KP6): 클릭 집계 전 "인기" 표기 배포 금지 |
| 5 | 배포(distribution) 계획 부재가 최대 구멍 — 채널-연락 수-게시물 수가 0 | 2~3장의 시딩 자산 + 5채널 플레이북 |

사실 검증 결과(리뷰 주장 직접 확인):
- **눈누마켓 실재 확인** — "유료폰트는 눈누가 못 오는 영역" 가정 폐기, 마스터플랜 3-2에 정정 기록.
- **AdSense "100페이지" 요구는 근거 없음** — 공식 기준은 독창-유용 콘텐츠. 대량 유사 페이지는 오히려 스팸 판정 리스크.
- **낸내 사례** — 폰트 284종을 갖추고도 월 사용자 1,000명 미만(2021, 긱뉴스). "폰트 수 = 수요"가 아니라는 실증. 카탈로그 패리티(KP3)는 유입 수단이 아니라 이탈 방지 수단으로 취급한다.

⚠️ 한계: 아래 채널별 유입 추정치는 팔로워 0 신규 브랜드 기준의 추정이며 실측이 아니다. 공개 후 주간 실측으로 대체한다.

## 2. 공개 전 30일 (D-30 ~ D-0): 2트랙

### 트랙 A — 제품 (AI 개발, 로드맵 순서 유지)

슬라이스2 검색 → 슬라이스3 클릭 집계 → 라이선스 신뢰 블록 데이터 100% → 컬렉션 10개 → 측정(GA4-서치콘솔-서치어드바이저) → 필수 문서 4종 → 공유 카드 기능(트랙 B W3에서 스펙 확정). 공개 게이트는 마스터플랜 17장 체크리스트.

### 트랙 B — 시딩 자산 (창업자, 주 5시간 x 4주 = 20시간)

| 주차 | 작업 (5h) | 산출물 |
|---|---|---|
| W1 | 시딩 대상 리스트 60곳 작성: 폰트 제작사 20(공동 홍보 후보), 디자인 커뮤니티 15(네이버 카페-오픈채팅-디스코드, 각각 홍보 규정 확인), 소형 계정-뉴스레터-교육자 25. 긱뉴스-디스콰이엇 계정 생성 + 사전 활동(타인 글에 유의미한 댓글) | 시딩 리스트 시트 |
| W2 | 위성 네이버 블로그 개설 + 글 2건(컬렉션과 1:1 짝). 스레드(Threads) 익명 계정 개설, "오늘의 폰트" 포맷 실험 3건 | 블로그 2건, 스레드 3건 |
| W3 | 제작사 접촉 문안 작성 — **"저희가 정리한 귀사 폰트 라이선스 요약이 정확한지 검수 부탁드립니다. 괜찮으시면 귀사 채널 소개도 부탁드립니다"** — 검수와 공동 홍보 의도를 최초 연락에서 투명하게 명시(숨긴 접근 금지). 1차 10곳 발송. 공유 카드 기능 스펙 확정 → 트랙 A 전달 | 발송 10건, 스펙 1건 |
| W4 | 출시 글 3종 초안: 긱뉴스 Show용(만든 과정 중심), 디스콰이엇용(제품 소개), 커뮤니티용(피드백 요청 프레임). 커뮤니티 관리자 사전 허락 5곳 확보. 제작사 2차 10곳 발송(누적 20곳 — 3장 채널 1의 전제 충족) | 출시 글 3종, 허가 5곳, 발송 누적 20건 |

공개 데드라인: D-30 기점 **45일 안에는 반드시 공개**한다 — 준비가 미달인 기능은 잘라내고 공개(성공 판정이 "공개 후 30일" 기준이므로, 공개 연기는 판정의 무한 연기다). 채널 5가 의존하는 공유 카드 기능이 미완성이면 채널 5는 "수동 캡처 이미지 게시"로 축소 실행한다.

## 3. 공개 후 30일 (D+1 ~ D+30): 5채널 플레이북

목표 합계: 중복 제거 후 500~1,200명. 1,000명은 기본값이 아니라 상위 실행 결과다.

| 순위 | 채널 | 실행 | 예상 유입 | 리스크와 완화 |
|---|---|---|---|---|
| 1 | 폰트 제작사 공동 홍보 | 검수 요청한 20곳 중 3~4곳의 공유 확보. 제작사별 전용 소개 페이지 + 공유용 이미지 제공 | 250~500 | 무응답 다수 — 검수 프레임 + 완성도 높은 페이지로 완화 |
| 2 | 긱뉴스 Show → 디스콰이엇 순차 출시 | "폰트 비교-라이선스 확인 도구"로 소개(아카이브가 아니라 도구로). 긱뉴스 먼저, 피드백 반영 후 2~3일 뒤 디스콰이엇. 반복 홍보 글 금지 규정이 있으므로 채널당 한 번의 강한 출시 글만 | 120~300 | 반응 없으면 재시도 불가 — 1차(긱뉴스) 피드백을 2차(디스콰이엇)에 반영해 승부 |
| 3 | 허가형 커뮤니티 배포 | 사전 허락받은 5곳에 "피드백 요청" 게시 | 80~180 | 셀프 프로모션 반발 — 허가 선행 + 피드백 프레임으로 완화 |
| 4 | 소형 계정-교육자 제휴 | "상업용 무료폰트 라이선스 표" 완성 자료를 25곳에 무상 제공 → 2~4회 소개 유도 | 180~400 | 거절률 50%+ — 자료 자체의 유용성으로 승부 |
| 5 | 스레드 + 공유 카드 | 주 3회 폰트 콘텐츠(비교-캔버스 결과 캡처) + 방문자가 결과를 이미지/고유 주소로 공유하는 카드 기능 | 50~150 | 계정 성장은 1~3개월 소요 — 팔로워가 아니라 단건 확산을 노림 |

광고 라이트 정책 연동: 공개 후 30일은 광고 0~1개(마스터플랜 8장). 첫인상 구간에서 "눈누보다 가벼운 화면"을 체감시키는 것이 KP0의 실행이다. AdSense 신청은 색인 시작 후.

측정 계약(의무): 외부에 게시하는 모든 링크에 UTM 파라미터(유입 경로 식별용 주소 태그)를 붙인다 — `utm_source=채널명, utm_medium=post, utm_campaign=launch30`. **UTM 없는 링크는 배포 금지.** 채널별 기여 측정과 4장의 재배분-경보가 전부 이것에 의존한다.

공개 후 주간 공수표 (주 5시간 상한):
- D+1주: 출시 실행 3h(제작사 공유 확인 + 긱뉴스 게시-댓글 응대) + 허가 커뮤니티 게시 1h + 스레드 1h
- D+2주: 디스콰이엇 출시-응대 1.5h + 교육자-소형 계정 자료 발송 2h + 스레드 1h + 지표 점검 0.5h
- D+3~4주: UTM 기여 상위 2채널에 3.5h(70%) 집중 + 나머지 1h + 지표 점검 0.5h. **기여 하위 2채널은 활동 중단**(버리는 규칙 — 5채널 전부를 주 5시간에 계속 굴리는 것은 불가능)

## 4. KPI와 킬 스위치

- 용어 정의: **"방문" = GA4 신규 사용자**. **"공유" = 제3자(제작사-커뮤니티-소형 계정)가 자신의 공식 채널에 올린 FontAgit 소개-링크 게시물** — 우리가 올린 출시 글과 방문자의 공유 카드 사용은 공유로 세지 않는다.
- 전면 킬 스위치(AND, 의도적): D+14에 신규 사용자 누적 250명 미만 **그리고** 공유 2건 미만 → 포지셔닝-채널 전면 재검토(Codex 처방 채택). 둘 다 실패일 때만 전면 재검토를 발동한다.
- 채널 경보(2층): D+14에 채널별 UTM 기여가 3장 예상 하한의 50% 미만이면 **그 채널만** 전술 수정 또는 중단 — 전면 재검토와 별개로 부분 실패를 조기 감지.
- D+30 판정: 누적 1,000명 미달 → 재배분 규칙 적용 후 60일 연장 1회. **재배분 규칙: UTM 기여 상위 2채널에 주간 시간 70%, 하위 채널 중단.** 연장 후에도 미달이면 마스터플랜 15장 킬 조건 절차로.
- 보조 KPI(제품 사용, GA4 이벤트): 검색 실행률(방문자 중 검색 사용 비율), 라이선스 원문 링크 클릭률, 재방문율. 방문이 늘어도 이 셋이 바닥이면 채널이 아니라 제품 문제다(KP1-KP2 검증 지표).
- 주간 기록: 신규 사용자, UTM 채널별 기여, 체류시간, 재방문율, 검색 실패어(마스터플랜 F-17 서버 로그로 수집 — GA4로는 수집 불가). GA4 + 서치콘솔.

## 5. 미확정과 한계

1. 눈누 폰트 총수 실측 필요 — 카탈로그 패리티(KP3)의 30/90일 목표 수치 확정용.
2. 채널 유입 추정치 전부 미검증 — D+7부터 실측으로 교체.
3. 제작사 공동 홍보와 익명 운영의 긴장 — 개인 노출 없이 브랜드(FontAgit) 명의로 접촉, 사이트 완성도와 문안 품질로 신뢰 보완.
4. RPM 실측 전까지 수익 전망(마스터플랜 2장)은 하단 가정(₩300~600)으로 읽을 것.
