# 문의 페이지 UI 개선 설계

## 목표

문의 페이지에 아이콘만 남는 오류를 없애고, 모든 문의 경로를 `contact@fontagit.com`으로 통일한다. 사용자가 이메일 주소를 바로 확인하고 큰 클릭 영역으로 메일 앱을 열 수 있게 한다.

## 원인

현재 세 문의 영역은 이메일 주소가 주석으로만 남아 있다. 주석은 화면에 표시되지 않으므로 봉투 이모지만 작은 단독 요소로 렌더링된다.

## 승인된 UI

- 일반 문의, 저작권 신고, 피드백 영역마다 같은 이메일 버튼을 표시한다.
- 이모지 대신 장식 없는 봉투 SVG 아이콘을 사용한다.
- 버튼에는 `contact@fontagit.com`을 글자로 표시하고 `mailto:contact@fontagit.com`으로 연결한다.
- 각 문의 영역은 얇은 테두리와 절제된 배경을 가진 카드로 구분한다.
- 버튼 전체를 클릭할 수 있게 하고, 키보드 초점 표시와 충분한 명도 대비를 유지한다.
- 모바일에서는 버튼이 카드 너비에 맞게 늘어나고 이메일 주소가 잘리지 않게 한다.

## 구조

페이지 내부에 작은 이메일 링크 표현을 한 번 정의하고 세 문의 영역에서 재사용한다. 별도 서버 저장이나 문의 폼은 추가하지 않는다. 응답 시간 영역은 안내 정보이므로 이메일 카드와 구분되는 간단한 안내 블록으로 유지한다.

## 동작과 예외

- 이메일 버튼을 누르면 사용자 기기의 기본 메일 앱이 열린다.
- 메일 앱이 연결되지 않은 환경에서도 주소 문구를 선택하거나 복사할 수 있다.
- 자바스크립트 없이도 `mailto:` 링크가 동작해야 한다.

## 검증

- 세 이메일 링크의 표시 문구와 `href`가 모두 `contact@fontagit.com`인지 확인한다.
- 작은 단독 이모지가 남지 않았는지 확인한다.
- 데스크톱과 모바일에서 카드 간격, 줄바꿈, 클릭 영역, 키보드 초점을 브라우저로 확인한다.
- 단순 표시 UI이므로 새 단위 테스트는 만들지 않는다. 기존 린트와 프로덕션 빌드를 실행한다.

## 제외 범위

- 웹 문의 폼
- 이메일 서버 또는 저장 API
- 자동 답장과 스팸 방어
- 문의 유형별 별도 이메일 주소
