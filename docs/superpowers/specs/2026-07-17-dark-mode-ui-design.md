# 애드블록 배너 및 글자 지원 검사 다크모드 개선 설계

## 목표

- 다크모드에서 애드블록 안내와 글자 지원 검사 영역이 흰색으로 표시되는 문제를 고친다.
- 애드블록 안내를 본문 흐름에서 분리해 작고 자연스러운 알림으로 표시한다.
- 기존 FontAgit 색상 체계를 그대로 사용해 라이트·다크모드가 함께 동작하게 한다.

## 확인된 원인

- 애드블록 CSS는 프로젝트에 없는 `--surface-secondary`, `--border-secondary`, `--text-secondary` 변수를 사용한다.
- 글자 지원 검사 CSS는 프로젝트에 없는 `--color-*` 변수를 사용한다.
- 변수를 찾지 못하면 CSS에 적힌 흰색 기본값이 적용되어 다크모드가 깨진다.
- 애드블록 배너가 전역 레이아웃에서 본문과 푸터 사이에 있어 전체 너비의 콘텐츠처럼 보인다.

## 적용 범위

### 1. 애드블록 안내

- 프로젝트 공통 변수인 `--surface`, `--surface-2`, `--border`, `--ink`, `--sub`, `--point`를 사용한다.
- 데스크톱에서는 화면 우측 하단의 작은 고정 알림으로 표시한다.
- 최대 너비를 제한해 긴 가로 띠가 되지 않게 한다.
- 푸터나 본문 높이에 영향을 주지 않는다.
- 모바일에서는 좌우 여백을 두고 화면 하단 메뉴 바로 위에 표시한다.
- 닫기 버튼은 마우스 올림, 키보드 초점, 다크모드에서 모두 분명하게 보이게 한다.
- 알림이 다른 핵심 조작을 가리지 않도록 적절한 `z-index`와 하단 간격을 둔다.

### 2. Playground 글자 지원 검사

- `GlyphCheckerSection`과 `GlyphChecker`의 존재하지 않는 `--color-*` 변수를 모두 공통 색상 변수로 교체한다.
- 바깥 영역, 폰트 선택창, 검사 카드, 입력창, 버튼, 결과 카드가 다크모드에 맞게 바뀌도록 한다.
- 경고·오류·성공·미지원 상태는 배경만 어둡게 만드는 것이 아니라 글자 대비도 함께 확보한다.
- 현재 기능과 문구, 검사 로직은 변경하지 않는다.
- 작은 화면에서도 입력창과 버튼이 기존처럼 세로 배치되도록 유지한다.

## 접근성

- 텍스트와 배경이 다크모드에서도 충분히 구분되게 한다.
- 닫기 버튼, 선택창, 입력창, 버튼의 키보드 초점 표시를 유지한다.
- 색상만으로 성공·실패를 구분하지 않고 현재의 글자와 기호를 함께 유지한다.

## 변경 예상 파일

- `apps/web/components/AdBlock/AdBlockBanner.module.css`
- `apps/web/components/GlyphCheckerSection.module.css`
- `apps/web/components/GlyphChecker.module.css`
- 필요할 때만 `apps/web/components/AdBlock/AdBlockBanner.tsx`

## 제외 범위

- 애드블록 감지 방식과 노출 조건 변경
- 광고 정책 또는 광고 슬롯 구조 변경
- 글자 지원 검사 로직 변경
- 새로운 전역 테마 시스템 도입

## 검증 방법

- `/trends`와 `/playground`를 라이트·다크모드에서 직접 확인한다.
- 데스크톱과 모바일 화면에서 배너 위치와 콘텐츠 가림 여부를 확인한다.
- 애드블록 닫기 버튼과 글자 검사 입력·선택 동작을 확인한다.
- UI 전용 변경이므로 새 단위 테스트는 만들지 않는다.
- 기존 린트와 빌드를 실행한다.
