## 2. 블로커 및 해소 절차

### B1. dev DB 조회 도구 장애 (여전히 유효)
- 증상: `supabase-dev` MCP query → `self-signed certificate` 에러. `supabase-prod`는 정상.
- **B1 판정 기준**: 시드 실행 = 비차단(파이프라인 코드가 직접 수행, 실제로 눈누 1110종 적재 성공). dev 현황 조회-검증만 이 도구가 필요.
- 해소: MCP dev 호스트-인증서 체인 점검(dev는 클라우드라 정상 cert여야 함). **TLS 검증 우회 금지**(sslmode=disable 등 쓰지 않고 근본 원인 해결). 우회 대안은 `.env.sandbox` psql 읽기.

### B2. 공유 워킹트리 — 여러 세션 동시 작업
- 실측: 이 세션 중 워킹트리 브랜치가 develop → feature/noonnu-tier-b-enrich → main 으로 계속 플립됨. 여러 세션이 같은 워킹트리를 공유.
- 해소: 코드 작업은 **별도 git worktree** 또는 develop 기준 **기능 브랜치**. develop 직접 병렬 커밋 지양(플립 유발). 매 작업 전 `git branch --show-current` 재확인. **내가 만든 파일만 스테이징**. 최종 병합은 담당 세션 1개.

## 3. 미완 이슈별 실행 계획

### #28 컬렉션 10개 + Tier C (MUST, high) — 폰트 수는 초과 달성
- 현황: 폰트 published 1240(목표 200 초과). **남은 것: 컬렉션 3→10, Tier C 무료대안 0→30, 필수필드 검증.**
- 단계: (1) 컬렉션 10개 주제 정의(손글씨/고딕/명조/디스플레이/본문/제목 등) + 폰트 매핑 (2) Tier C 무료대안 수집(유료폰트 대안 매핑) (3) dev 시드(B1 무관, 파이프라인 직접) → 필수필드 결측 0 검증.
- 완료 게이트(품질 기준): 컬렉션 10개(각 최소 5종), 중복 0, 필수필드 결측 0. 필수필드(fonts NOT NULL): slug, name_en, source_tier, category_ko, subsets, variants, weights, is_commercial_free, license_verified, official_url.
- dev→prod 승격-롤백: dev 시드-검증 → 사용자 승인 → prod 백업 → prod 반영(쓰기 사용자 확인) → SSG 재빌드-배포 → 화면 확인. 실패 시 시드 변경분만 롤백 + 이전 배포본 복원.
- 조율: 눈누 적재는 완료. 컬렉션/TierC는 상대적으로 독립. 진행 세션과 DB 쓰기 단일화.
