## 2. 블로커 및 해소 절차 (실행 전 반드시 처리)

### B1. dev DB 조회 도구 장애
- 증상: `supabase-dev` MCP query → `self-signed certificate in certificate chain`. `supabase-prod` MCP는 정상.
- 영향: #28 시드 대상(dev)의 현재 폰트/컬렉션 수 파악 불가 → 시드 착수-검증 불가.
- 해소 후보(택1, 확인 필요):
  1. MCP 설정에서 dev 접속 호스트-TLS 옵션 점검(dev는 supabase.com 클라우드여야 정상 cert. self-signed면 prod 자체호스팅 쪽으로 잘못 라우팅됐을 가능성).
  2. 우회: `.env.sandbox`의 dev 접속정보로 psql 직접 조회(읽기).
- 검증: `select count(*) from fontagit.fonts;` 가 dev에서 정상 반환.

### B2. 공유 워킹트리 — 다른 세션 활발 커밋
- 증거: 방금 `d7b6f12 fix: 눈누 임포터 기존 폰트 조회 maybe_single 수정`이 develop에 커밋됨(#28 핵심 파일 `noonnu_import.py`). 다른 세션이 #28 시드를 진행 중.
- 위험: 같은 develop 워킹트리에서 병렬 코드 작업 시 브랜치 플립 → 내 변경이 엉뚱한 브랜치로. 같은 데이터 중복 삽입.
- 해소 규칙:
  - #28은 **다른 세션에 위임**(중복 실행 금지). 내가 손대기 전 조율 확인.
  - 내가 코드 작업 시 **별도 git worktree**(`.worktrees/`)로 격리. 매 작업 전 `git branch --show-current` 재확인.
  - **내가 만든 파일만 스테이징**(전역 `git add .` 금지).

## 3. 미완 이슈별 실행 계획

### #28 폰트 200종 + 컬렉션 10개 (MUST, priority: high)
- 목표: A(한글+라틴) 100 + B 50 + C 30 = 200종, 컬렉션 10개, 필수필드 100%.
- 현황: prod published 130종 / 목표 200 → **약 70종 부족**. 컬렉션 3 → 10 → **7개 부족**. (dev 현황은 B1 해소 후 확정)
- 단계:
  1. B1(dev DB 조회) 해소 → dev 현재 폰트/컬렉션/필드 결측 실측.
  2. 부족분 소스 확정: A/B는 눈누 드래프트 임포터(PR #52 계열, `noonnu_import.py`) 후속 수집, C(무료 대안 Tier)는 구글폰트 등.
  3. 컬렉션 10개 주제 정의(예: 손글씨-고딕-명조-디스플레이-귀여운-레트로-본문용-제목용-라운드-모노). 각 컬렉션에 폰트 매핑.
  4. dev 시드 실행 → 필수필드(이름-라이선스-subset-미리보기) 100% 검증.
- 검증: `select count(*) from fontagit.fonts where status='published'` = 200, `collections` = 10, 결측 필드 0.
- 의존/조율: **다른 세션이 진행 중(B2)** → 조율 후 착수. B 에픽(#60/#56)이 이 시드에 의존.

### #57 /fonts 총 개수-메타 + 페이지네이션 (SHOULD, medium) — 즉시 착수 가능
- 목표: /fonts에 총 폰트 수-메타 표시 + 페이지네이션(또는 무한스크롤). **#28 비의존.**
- 제약: 웹은 `output: 'export'` SSG → 서버 페이지네이션 없음. 빌드타임 정적 분할 or 클라이언트 슬라이싱 검토.
- 단계: 총개수 메타 컴포넌트 → 정적 페이지 분할 전략 결정(페이지당 N개 pre-render) → UI + 회귀 테스트.
- 검증: /fonts 총개수 정확, 페이지 이동 동작, 빌드 성공. worktree 격리 필요(B2).

### #53 메인/헤더 검색 통일 (검색 UX, medium) — 즉시 착수 가능
- 목표: 메인 검색과 헤더 검색 컴포넌트-동작 통일. **#28 비의존.**
- 현황: 완료조건 이슈 body에 명시없음 → 착수 전 "통일" 범위 정의 필요(같은 컴포넌트 재사용? 동일 자동완성?).
- 단계: 현재 두 검색 구현 비교 → 공통 컴포넌트로 통합 → 회귀 테스트(#51 검색 회귀 계열 재사용).
- 검증: 두 진입점 동작 일치, 자동완성 회귀 green. worktree 격리.

### #56 컬렉션 페이지 허브화 (에픽 #60, medium) — #28 의존
- 목표: 컬렉션 페이지를 내부 링크 허브로(SEO 키워드 기반).
