## 결정 사항 (Decisions)

| # | 결정 | 근거 |
|---|------|------|
| 1 | rate limit 2계층: A(Kong IP 분당30) + B(DB 폰트별 10초20건) | 위협모델 "둘 다"(랭킹조작+리소스보호). 정적 SSG라 Next 서버 프록시 불가 |
| 2 | B는 blocking 아닌 `pg_try_advisory_xact_lock`(2-key namespace) | PR 듀얼리뷰 S2: blocking은 봇 폭주 시 연결 점유. try는 획득 실패 시 무시 → 리소스 보호 |
| 3 | 마이그레이션 rate limit=0008, 등록폼=0009 재배치 | 순번 연속성 |
| 4 | prod 쓰기 = `supabase_admin` role + SSH docker exec | `postgres`는 superuser 아니고 fontagit 스키마 권한 없음(permission denied). 소유자=supabase_admin |
| 5 | prod 배포는 스키마만 먼저, 폰트/Kong 별도 | 공유 인프라 리스크(특히 Kong 공유 게이트웨이) |
| 6 | dev MCP self-signed 블로커 → psql `sslmode=require` 우회 | 실증됨 |
