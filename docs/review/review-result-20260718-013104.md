## 1. Codex 리뷰 요약

### 핵심 피드백 (12건 요약)
1. 실행 위치 확정 — OCI VM systemd timer/cron 권장(prod DB 근접). GitHub Actions는 예약 지연-비활성화 위험 → 보조용.
2. 배치 전용 DB 권한 — 웹 anon INSERT 재사용 금지, service_role 전권 금지. 수집 테이블/저장 함수 한정 계정.
3. 법적 검토를 행위 단위로 분해 — 메타수집/파일다운로드/재배포/미리보기/가격표시 각각 허용-금지-미확인. MVP는 메타+공식링크만, 폰트 파일 재호스팅 제외.
4. 공식 공공 폰트 출처 실재(웹검색 확인): 한국저작권위원회 공유마당 안심글꼴, 한국문화정보원 공공누리 글꼴, 공공데이터포털 추천 글꼴(98건, 1회성 → 교차검증용).
5. 라이선스를 무료/유료 2값으로 저장 금지 — 공공누리 유형(출처표시/상업/변경), 임베딩-CI/BI-판매 제한을 필드별로.
6. 기관 신규 폰트 발견 2단계 — 구조화 수집(공유마당/공공누리/공공데이터) + 발견 작업(기관 홈페이지/보도자료/RSS/사이트맵). 검색결과는 자동등록 금지, 후보 큐 경유.
7. 출처 우선순위는 레코드 전체가 아닌 필드별(라이선스=권리자 공식, 가격=판매처, 눈누=발견/보조). 충돌 시 덮어쓰지 말고 양쪽 원본-확인일-충돌상태 보존.
8. 중복 모델 선설계 — 패밀리 vs weight/style 파일 분리. 해시는 동일파일만, 패밀리 병합은 정규화명+제작사+OpenType 내부명+공식URL. 애매하면 자동병합 금지.
9. 증분에 신규뿐 아니라 변경-삭제 포함 — 소스ID/최종확인시각/콘텐츠지문/ETag. 1회 누락 즉시삭제 금지, 연속 미발견 시 비활성 후보.
10. Silent failure 기준을 완료조건에 — 0건-급감-필수필드 누락-선택자 실패는 성공 처리 금지, 알림.
11. 정적 사이트 반영 의미 명확화 — 런타임 Supabase 조회면 DB 반영=노출, 빌드타임 포함이면 재빌드/재배포 필요.
12. 완료조건을 검증 가능하게 — 멱등성, 신규/변경/삭제 처리, 4회 연속 예약 성공, 원본URL-수집일-라이선스근거, 급감 알림, 애매항목 비공개, 반영→노출 목표시간.

### 개선 제안(우선순위)
- 최우선: Epic 전환 + MVP를 "눈누 증분 + 공유마당/공공누리 메타"로 제한, OCI VM 전용 배치, 파일/가격 제외, 법적 허용표-배치 DB 권한-출처 스키마 선설계, "중복 없음"→"확실 병합/애매 대기".
- 다음: 수집 이력-원본 관측 저장, 눈누 수집기 멱등-증분-실패감지 보강, 2번째 소스 연결, dev 검증 후 prod, 정적 노출 방식 검증.
- 후순위: 기관 발견기, 기업/비영리 출처 관리, 유료는 판매처별 offer 모델, 영어/상용은 별도 Epic.
