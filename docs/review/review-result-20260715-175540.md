## 2. Claude 크로스 리뷰

### 종합 소견
Codex 리뷰는 환각-과잉해석 없이 16건 모두 사실 기반이거나 방향이 타당하다. 특히 P0 5건(입력보안 RPC, Top10 조회원천, 재빌드정책, 스키마노출, 남용대응)은 코드/스키마로 실재가 확인된 구현 차단급 이슈다. 일부는 기획서가 이미 내린 결정(클릭 어뷰징 초기 미방어)이라 "수용된 리스크 명시"로 처리하면 된다. 동의하지 않는 항목은 없다.

### 동의하는 핵심 피드백 (Top 3)
1. **입력 보안 RPC화(3)**: anon insert만으로는 status-시간 조작이 열려 있다. published 상태 주입은 미검수 폰트 노출로 직결되는 보안 이슈. 클릭/등록을 SECURITY DEFINER RPC로 감싸 허용 컬럼만 받고 상태-시간은 DB가 강제해야 한다.
2. **Top10 조회 원천 명확화(2)**: 스펙이 "빌드타임 생성"을 안 적어 daily/raw 모순처럼 보인다. get_top_fonts RPC(집계 전용, 원본 미노출)로 확정하면 모순도 보안도 해결.
3. **정적/검색 데이터 시점(1)**: 재빌드 정책이 없으면 검색결과가 아직 없는 상세로 404. 데이터 갱신=재빌드 단일 경로로 못박아야 한다.

### 동의하지 않는 피드백
없음. 16건 모두 사실 기반 또는 방향 타당. 단 #4는 기획서가 이미 "초기 미방어"로 결정한 부분이라 신규 방어 구축이 아니라 결정 명시로 처리.

### Codex가 놓친 추가 관점
- **유료 폰트 화면이 실데이터에서 전부 빈다**: published 제약(무료 라이선스만)으로 tier='paid'가 생성 불가. 필터 "유료" 탭, 유료 상세, F-13 "무료 대안 3개"가 실데이터에선 표시할 대상이 없다. 이번 세션에서 "유료=데이터 없음"을 인지하고 빈 상태 처리하거나 Tier C 스코프를 명시적으로 후속으로 미뤄야 한다.
- **정규화 SSoT 부재**: 파이프라인(uploader.py)과 검색이 각자 정규화하면 필연적으로 어긋난다. DB 정규화 함수 하나를 단일 출처로 삼고 파이프라인-검색이 공유해야 한다.

## 3. 최종 권고사항

### 즉시 반영 (Must — P0, 구현 전 스펙 보완)
- **입력 보안 RPC화**: `record_click(font_id)`, 등록 `submit_font(...)`를 SECURITY DEFINER RPC로. status-시간은 DB 강제, 원본 클릭 테이블은 anon 미노출.
- **Top10 조회 확정**: 빌드타임 생성 + `get_top_fonts(period, limit)` RPC 단일 경로. 초기 raw 직접조회 문구 삭제.
- **재빌드 정책**: 데이터 갱신=재빌드가 유일 경로임을 명시. 검색결과는 빌드된 slug와 정합(미존재 시 상태 페이지).
- **fontagit 스키마 노출**: PostgREST exposed schema + `.schema('fontagit')` 설정을 구현 전 확인에 추가.
- **매핑 정정**: tier는 is_commercial_free 기반, source_tier 별도. 유료 폰트 부재(빈 상태) 처리 명시.

### 검토 후 반영 (Should — P1)
- 검색 RPC 입출력 계약(반환필드-최대수-동점-빈검색어-특수문자-debounce).
- 클릭/등록 테이블 타입-제약-인덱스-중복기준-보관정책 표.
- async/await 전환 호출부 정리("화면 무변경" 문구 정정).
- 정규화 규칙 DB 함수로 SSoT 통일(파이프라인 공유).
- 클릭 fire-and-forget + 이동 보장, 등록폼 상태모델 재정의, SEO 산출물 완료기준 추가.

### 참고 (Nice to have — P2)
- RLS 권한 테스트(anon update/delete/raw-select 차단, 미공개 노출 방지) + 정적 export 전체 빌드 검증.
- 클릭 원본 보관기간-일별 롤업 전환 조건-검색/제출 실패 관측 방법.
