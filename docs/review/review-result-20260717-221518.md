### 구체적 피드백

1. **SVG 접근성 미명세 (High).** "장식 없는 봉투 SVG"라면 `aria-hidden="true"`로 두고 접근 이름은 링크 텍스트(`contact@fontagit.com`)가 제공해야 한다. 명시가 없으면 스크린리더가 "이미지 + 이메일"을 중복 낭독할 수 있다.

2. **회귀 방지 장치 부재 (High).** 이번 버그의 원인은 "이메일이 주석으로만 존재"인데, 검증을 브라우저 육안 확인에만 의존한다. 세 개의 `mailto:contact@fontagit.com` 링크가 렌더되는지 확인하는 최소 렌더 테스트 1개면 동일 회귀를 코드로 막는다(값 대비 비용이 매우 낮음). 이 컴포넌트가 db 모듈을 import하면 테스트에서 db mock이 필요하다(웹 테스트 env 취약성 주의).

3. **단일 주소 통일 시 유형 라우팅 손실 (Medium).** 일반/저작권/피드백을 한 주소로 모으면 수신 측에서 분류가 어렵다. `mailto:contact@fontagit.com?subject=[저작권 신고]`처럼 유형별 subject 프리필을 넣으면 "유형별 별도 주소" 없이도 분류가 가능하다 — 제외 범위를 건드리지 않는 저비용 개선.

4. **스팸 노출 트레이드오프 미기재 (Medium).** 정적/SSG 페이지의 평문 `mailto`는 봇 수집 대상이다. "스팸 방어 제외"는 정당한 결정이지만, 트레이드오프를 문서에 한 줄 남겨 의도된 선택임을 밝히는 편이 좋다.

5. **줄바꿈 기법 미명시 (Low).** "주소가 잘리지 않게"는 목표만 있고 수단이 없다. 좁은 화면 대비 `overflow-wrap: anywhere` 등 기법을 명시하면 구현 편차를 줄인다.

6. **추적성 (Low).** 대상 파일/컴포넌트 경로와 응답 시간 블록 실제 문구를 문서에 적어 문서-코드 동기화를 명확히 한다.

### 개선 제안 (우선순위순)

1. (필수) 봉투 SVG `aria-hidden="true"` + 링크 접근 이름 확보를 설계에 명시.
2. (필수) 세 `mailto` 링크 존재를 검증하는 최소 렌더 테스트 1개 추가 — "무테스트" 방침의 예외로 다룰 가치가 있음.
3. (권장) 유형별 `?subject=` 프리필로 단일 주소 라우팅 보완.
4. (권장) 스팸 수집 노출 트레이드오프를 제외 범위에 한 줄 기재.
5. (선택) 줄바꿈 기법·대상 파일 경로·응답시간 문구를 문서에 구체화.

## 2. Codex 크로스 리뷰

### 종합 소견

접근성, 모바일 줄바꿈, 기존 주석 제거 지적은 실제 구현에 바로 반영할 가치가 있다. 다만 단순 표시 UI에 렌더 테스트를 추가하라는 제안은 이 저장소의 사용자 테스트 원칙과 충돌하며, Contact 페이지는 DB 모듈을 사용하지 않아 관련 mock 우려도 해당하지 않는다.

### 항목별 판정

| # | Claude 피드백 | 판정 | Codex 의견 |
|---|-------------|------|------------|
| 1 | SVG 접근성 명세 추가 | 동의 | 봉투는 장식이므로 `aria-hidden="true"`가 필요하고 링크 텍스트가 접근 이름을 제공해야 한다. |
| 2 | 최소 렌더 테스트 추가 | 동의하지 않음 | 사용자는 단순 UI 컴포넌트 테스트를 만들지 말라고 명시했다. 브라우저에서 세 링크와 모바일 UI를 직접 확인하는 편이 현재 정책에 맞다. Contact 페이지에는 DB import도 없다. |
| 3 | 문의 유형별 subject 프리필 | 부분 동의 | 같은 주소를 유지하면서 분류할 수 있어 유용하지만, 승인된 설계에 없던 동작이다. 사용자 선택 후 반영할 Should 항목이다. |
| 4 | 평문 이메일의 스팸 노출 기록 | 동의 | 방어 기능을 추가하진 않더라도 의도된 트레이드오프임을 문서에 남기는 것이 좋다. |
| 5 | `overflow-wrap: anywhere` 명시 | 동의 | 모바일에서 이메일 주소가 카드 밖으로 나가지 않는다는 승인 요구사항을 구체적으로 보장한다. |
| 6 | 대상 경로와 응답 시간 문구 명시 | 부분 동의 | 대상 파일 경로는 추적성에 유용하다. 응답 시간 문구는 기존 값을 변경하지 않으므로 설계에 그대로 복제할 필요는 없다. |

### 동의하는 핵심 피드백

1. 장식 SVG는 스크린리더가 중복 낭독하지 않도록 숨긴다.
2. 모바일 이메일 줄바꿈 규칙을 구체적으로 정한다.
3. 기존 이모지와 TODO 주석을 모두 제거한다고 명시한다.

### 동의하지 않는 피드백

렌더 테스트 추가는 이번 저장소의 테스트 범위를 벗어난다. 단순 UI에는 새 단위 테스트를 추가하지 않고, 실제 브라우저의 데스크톱·모바일 확인과 기존 린트·빌드로 검증한다.

### Claude가 놓친 추가 관점

- 현재 코드를 직접 확인한 결과, 세 위치 모두 이메일 주석 뒤에 이모지만 남는 동일 원인이다.
- 사용자의 다른 미완료 작업이 작업 트리에 있으므로 구현 시 문의 페이지와 설계 문서만 정확히 스테이징해야 한다.
- UI 규칙을 적용하기 전 `ui-skills`에서 실제 polish/accessibility 지침을 로드해야 한다.

## 3. 최종 권고사항

### 즉시 반영 (Must)

- 봉투 SVG에 `aria-hidden="true"`를 적용하고 이메일 텍스트로 접근 이름을 제공한다.
- 기존 봉투 이모지와 이메일 TODO 주석을 세 위치에서 모두 제거한다.
- 모바일 이메일 링크에 `overflow-wrap: anywhere`를 적용한다.
- 대상 파일을 `apps/web/app/contact/page.tsx`, `apps/web/app/contact/page.module.css`로 명시한다.

### 검토 후 반영 (Should)

- 일반 문의, 저작권 신고, 피드백별 메일 제목을 미리 채울지 사용자에게 확인한다.
- 평문 이메일이 스팸 수집 봇에 노출될 수 있음을 설계에 기록한다.

### 참고 (Nice to have)

- 기존 응답 시간 문구는 변경하지 않고 유지한다.
- 새 단위 테스트 대신 실제 브라우저에서 데스크톱·모바일 링크와 키보드 초점을 확인한다.
