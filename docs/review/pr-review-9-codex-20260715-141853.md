### High 이슈 - Should-fix

1. `apps/pipeline/src/fontagit_pipeline/licenses.py:75-81, 108-110`  
   **문제:** 정상 HTTP 응답에 `tree`가 없으면 조회 성공처럼 빈 결과 또는 일부 결과를 반환합니다.

2. `apps/pipeline/src/fontagit_pipeline/licenses.py:75-81, 108-116`  
   **문제:** JSON 파싱 실패만 감싸며, 올바른 JSON이지만 구조가 잘못된 경우는 방어하지 못합니다.

   **제안 수정:** `data`는 객체, `tree`는 배열, 각 항목은 객체인지 검사하고 모든 형식 오류를 `LicenseFetchError`로 통일하세요.

### 긍정적 관찰

- 폰트 upsert, 기존 alias 삭제, 재삽입이 하나의 DB 함수에서 실행돼 실패 시 함께 롤백됩니다.
- 빈 alias 배열도 기존 alias를 제거하므로 stale alias 제거 요구와 일치합니다.
- `SECURITY DEFINER`에 고정된 `search_path`를 사용하고 `PUBLIC`, `anon`, `authenticated` 권한을 회수한 점이 좋습니다.
- SQL 문자열 조립이 없어 SQL Injection 위험이 낮습니다.
- 첫 RPC 실패를 삼키지 않고 즉시 다시 던지며, 로그에 시크릿 대신 slug만 남깁니다.
- alias 정규화 중복 제거와 빈값 제거가 uploader 경계에서 명확하게 처리됩니다.
