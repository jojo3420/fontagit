### Critical 이슈 - Must-fix (머지 차단)

#### 1. prod에서는 컬렉션 아이템 시드가 영구히 비게 될 수 있음

- 파일:라인: `supabase/migrations/0003_seed_collections.sql:17-31`
- 문제: 현재 prod에는 폰트가 0건인데, 컬렉션 아이템 시드가 이미 존재하는 폰트와의 `JOIN`에 의존합니다. 마이그레이션이 먼저 실행되면 아이템 8건은 0건으로 처리됩니다. 이후 폰트를 적재해도 이미 끝난 마이그레이션은 다시 실행되지 않습니다.
- 근거(diff 인용): `join fontagit.fonts f on f.slug = v.font_slug`와 `on conflict ... do nothing`만 있고, 필요한 폰트가 없을 때 실패시키거나 나중에 재실행하는 장치가 없습니다.
- 제안 수정: 컬렉션 아이템 시드를 폰트 적재 후 실행하는 반복 가능한 seed 작업으로 분리하세요. 최소한 기대한 8건이 생성되지 않으면 `raise exception`으로 마이그레이션을 중단해야 합니다.

### High 이슈 - Should-fix (머지 전 수정 권장)

#### 1. 최신 등록순을 인기·클릭 순위로 잘못 표시함

- 파일:라인: `apps/web/lib/db/trends.ts:8-20`, `apps/web/app/trends/page.tsx:6-17`, `apps/web/app/page.tsx:6-12`
- 문제: 데이터는 `created_at DESC`의 최신 10건인데 화면에는 “이번 주 인기”, “이동 클릭 기준”이라고 표시됩니다. 사용자에게 사실과 다른 순위를 보여줍니다.
- 근거(diff 인용): `fonts.slice(0, 10)`과 `moves: font.moves`를 사용하면서 UI는 `이번 주 인기 폰트`, `이동 클릭 기준 인기 순위입니다`를 유지합니다.
- 제안 수정: 슬라이스3 전까지 “최신 등록 폰트”로 표시하고 주간·월간 탭을 숨기거나 비활성화하세요. 폰트 목록의 활성 정렬도 “인기순”이 아니라 “최신순”이어야 합니다.

#### 2. 운영 URL 누락 시 SEO가 가짜 도메인으로 정상 빌드됨

- 문제: 운영 환경변수가 빠져도 빌드가 성공하며 canonical과 sitemap이 `fontagit.example.com`을 가리킵니다. 검색엔진에 잘못된 주소가 등록될 수 있습니다.
- 근거(diff 인용): `process.env.NEXT_PUBLIC_BASE_URL || "https://fontagit.example.com"`이며 테스트도 실제 도메인이 아닌 `toContain("fontagit")`만 확인합니다.
- 제안 수정: production 빌드에서는 `NEXT_PUBLIC_BASE_URL`이 없거나 유효한 절대 URL이 아니면 실패시키세요. 끝의 `/`도 제거해 이중 슬래시를 방지해야 합니다.

#### 3. 데이터 이상을 빈 화면이나 부분 컬렉션으로 조용히 숨김

- 파일:라인: `apps/web/lib/db/fonts.ts:14-15`, `apps/web/lib/db/collections.ts:81-82,124-137`, `supabase/migrations/0004_fix_collections_anon_rls.sql:16-20`

### 긍정적 관찰

- `output:"export"`를 유지하고 API Route나 상주 서버 로직을 추가하지 않았습니다.
- Supabase 접근을 `lib/db`로 제한하고 `schema:"fontagit"` 설정을 정확히 적용했습니다.
- 목록 조회에서 aliases와 collection_items를 `.in()`과 `Map`으로 묶어 N+1 조회를 피했습니다.
- `FontKey | null`과 `familyOf()`를 도입하고 주요 렌더링 호출처를 함께 수정했습니다.
- 조회 오류를 대부분 다시 던지고, 실제 미존재에 해당하는 `PGRST116`만 `null`로 구분했습니다.
- anon key만 사용하고 service role 같은 비밀 키는 노출하지 않았습니다.
- 상세 메타데이터, canonical, sitemap을 실데이터 slug 기반으로 생성하는 방향은 적절합니다.
