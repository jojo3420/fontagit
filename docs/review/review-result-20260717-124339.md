### 동의하는 핵심 피드백 (Top 3)
1. 초성 기대값(1): 명백한 계산 오류. 테스트가 이 값을 쓰면 GREEN이 거짓이 됨. 즉시 수정.
2. lim 강제 + 오류/0건 구분(8, 12): anon 직접 호출-과부하-UI 오인의 실질 리스크. 함수 내부 clamp와 503/200 분리로 방어.
3. IME Enter + 하이라이트 대상(13, 15): 한글 조합 중 Enter 오작동, 별칭 검색 시 헛 하이라이트 — 실사용 첫 화면에서 바로 드러날 결함.

### 동의하지 않는 피드백
없음. 16은 "기존 RPC가 이미 처리"라 부분 동의(문서 명시로 흡수), 18은 정보성.

### Codex가 놓친 추가 관점
- Supabase(PostgREST) 함수 시그니처 변경 후 스키마 캐시 리로드(`notify pgrst, 'reload schema'`)를 배포 절차에 넣지 않으면 dev/prod에서 새 2인자 호출이 일시적으로 404가 날 수 있음 → 마이그레이션 말미에 포함.
