### 동의하는 핵심 피드백 Top 3
1. 라이선스 기본값(#1) — 잘못된 라이선스 공개는 기획상 AdSense 수익 파이프를 끊는 최대 리스크. 반드시 null/unknown.
2. 중복 제거(#2) + 라틴 100 절차(#3) — 수집 알고리즘의 정확성 핵심. 인기순 조회 후 한글 전체 추출, 그다음 라틴 100 추출, 그리고 family 합집합으로 고정.
3. 오류 처리 묶음(#5-#6-#7) — 타임아웃/재시도/원자적 저장/키 마스킹. "동작하는 스크립트"가 실제로 안전하게 도는 최소 조건.

### 동의하지 않는(부분) 피드백
- #10: "특수문자까지 안전하게 URL 인코딩"은 구글 specimen 규칙과 충돌. 구글은 공백을 `+`로 쓰므로 퍼센트 인코딩하면 깨진다. 공백→`+`만 유지, 나머지는 family가 ASCII 영숫자-공백 위주라 문제 없음. 방어적으로 비ASCII family가 나오면 로그 남기고 건너뛴다.

### Codex가 놓친 추가 관점
- 구글 API `subsets`에 `korean`이 명시되므로 한글 필터는 문자열 매칭으로 안정적. 다만 `variants`가 `regular`/`italic`/`700italic` 등 혼재하므로 정규화(숫자 weight + italic 플래그) 규칙도 짧게 정해두면 좋다.
- webfonts API는 페이지네이션 없이 단일 응답 → 페이징 처리 불필요(과설계 경계).
