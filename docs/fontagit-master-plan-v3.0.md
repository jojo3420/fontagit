## 1. 프로젝트 개요

- 제품: 국내외 무료-유료 폰트를 검색-비교하고 공식 다운로드/구매로 연결하는 웹 아카이브.
- 브랜드: FontAgit(폰트 아지트) — "폰트 덕후들의 아지트".
- 목적: 검색 유입 → 광고 수익. SEO와 페이지 속도가 최우선.
- 원칙: 폰트 파일을 직접 호스팅하지 않고 공식 페이지로 링크만 한다.
- 1차 사용자: 검색으로 유입되는 일반 사용자-디자이너-마케터-학생.
- 포지셔닝(2026-07-16 전환): 눈누와 같은 링크 제공 모델로 정면에서 겨루는 **정공법 품질 우위** — 광고는 더 가볍게, 정보는 더 정확하게, 검색은 더 편하게, 보유량은 동등 이상. 상세는 3장.

## 2. 수익 모델 현실 점검

RPM(페이지뷰 1,000회당 광고 수익)은 추정 가정이다(한국 일반 콘텐츠 ₩1,000~4,000). 폰트 카테고리는 고단가 분야가 아니므로 하단 가정도 각오한다. 운영 중인 다른 사이트 실측값이 있으면 보정한다.

| 월 페이지뷰 | 예상 월 수익(가정 RPM ₩1,000~4,000) |
|---|---|
| 100,000 | 약 10만 ~ 40만 원 |
| 300,000 | 약 30만 ~ 120만 원 |
| 1,000,000 | 약 100만 ~ 400만 원 |

의미 있는 부수입(월 30만+)에는 월 10만~30만 PV 필요. 6개월~1년짜리 SEO 장기전이며, 승인-색인 전 3~4개월은 수익 0 구간이다. 정적 사이트라 서버비가 사실상 0이므로(오라클 무료/보유 자원) 손익분기점도 0 — 리스크는 돈이 아니라 시간이다.

### 키워드 우선순위 (제로클릭 방어)
정보성 질문("○○체 상업적 이용 되나요?")은 검색 화면의 AI 요약이 답해버려 클릭이 안 난다. 방어가 되는 건 **클릭이 필수인 검색**이다:
1. 다운로드-획득 의도("지마켓산스 다운로드", "○○체 ttf") — 최우선
2. 폰트명 단독(내비게이션)
3. 모음-추천(컬렉션) — 검색량 크고 클릭 살아있음
4. 정보성(라이선스 질문) — 보조 섹션으로만

### 유입 채널: 구글 + 네이버
- 구글: 주력. 서치콘솔 등록, SSG로 속도 확보.
- 네이버: 서치어드바이저 + **위성 네이버 블로그**(모음 글 → 본 사이트 링크). 블로그 글 1건 = 컬렉션 1개와 짝.

### 경쟁 지형
눈누(무료 한글 폰트 장악. 눈누마켓으로 인디 유료폰트 판매 + 영어-일본어 페이지 운영 — 2026-07-16 직접 확인), 구글폰트(한국어 SEO 약함 — 틈), 네이버 잡글(정보 낡음 — 최신 확인일이 무기). 대응은 정면 회피가 아니라 정공법 품질 우위(3장). 보조 공략 3영역: ① 한/영 교차 검색어(지마켓 산스 = gmarket sans) ② 유명 유료폰트 정보 + 무료 대안 ③ 해외 유명 폰트 한국어 페이지.

솔직한 한계: 알리아스-라이선스 요약은 눈누가 몇 주면 따라한다. 초기 해자는 없고, 해자는 **콘텐츠 축적 속도 + 트렌드 데이터 누적**에서 나온다. 발행을 멈추면 지는 게임이다.

## 3. 서비스 정의 + 차별점-킬링 포인트

> 찾는 폰트가 무료든 유료든, 한글 이름이든 영문 이름이든, 한 번의 검색으로 정확한 폰트와 최신 라이선스 정보, 공식 경로, 그리고 지금 뜨는 트렌드까지 보여주는 폰트 아지트.

### 3-0. 포지셔닝 전환 — 정공법 품질 우위 (2026-07-16 2차 개정)

v3.0 원안의 "눈누 정면 회피"를 폐기하고 **정공법 품질 우위**로 전환한다. 같은 링크 제공 모델로 눈누와 정면에서 겨루되, 사용자 경험의 모든 축에서 "동일하거나 더 낫게"를 만든다.

- **광고**: 눈누보다 적게. 눈누의 높은 광고 밀도는 확인된 사용자 불만 지점이다. 성장기(일 방문 500 미만)는 광고 0~1개, 이후 단계 확대(8장).
- **정보**: 동일하거나 더 정확하게. 라이선스 확인일-원문 링크 즉답.
- **편리성**: 동일하거나 더 편하게. 현대적 디자인, 다크모드, 속도(CWV).
- **보유량**: 눈누가 제공하는 무료 폰트는 전량 커버(카탈로그 패리티, 단계 목표).
- **검색**: 더 편하게. 알리아스-초성-오타 교정.

전제와 리스크(솔직 버전): 품질 우위는 "방문한 사용자"에게만 작동한다. 첫 방문을 만드는 문제는 별도 그로스 플랜(`docs/fontagit-growth-plan-30d.md`)이 담당한다. 눈누 인지도 벽(16장)은 여전하며, 정공법은 회피 전략보다 콘텐츠-품질 유지 공수가 크다. 광고 라이트와 AdSense 수익 목표의 충돌은 "성장기 최소 광고 → 트래픽 확보 후 확대" 순서로 해소한다.

| **C** | 유명 유료폰트 | 수동 | 30 | 없음(텍스트) | **공식 구매 페이지만** | 가격-라이선스 개요 |
| **D** | 창작자 제보 | 신청 폼 | Phase 2 | Tier B 기준 | 창작자 공식 페이지 | Tier B 기준 |

원칙: **수집은 대량, 공개는 선별**(고유 콘텐츠 채운 것만). **AI는 초안, 공개는 사람**(검수 없는 대량 생성은 구글 스팸 정책 대상 + 라이선스 오기 치명적).

### Tier B 크롤링 2단계 (확정)
- **1단계 — 눈누 seed만**: 폰트명-제작사-공식 URL이라는 사실만 `review_queue`에 수집. 눈누의 설명-라이선스 요약은 복제 금지.
- **2단계 — 제작사 공식 사이트 세부**: 실제 라이선스-굵기-다운로드 경로는 1차 출처에서 확인. 회사마다 구조가 달라 파싱 가능분만 자동, 나머지는 사람 확인-승인.
- 지킬 선: robots.txt, 요청 속도 제한, 사실만 수집(콘텐츠 복제 금지), 파일 재호스팅 금지. 이 2단계 구조 자체가 리스크를 낮춘다.

## 6. 검색 설계

원리: 폰트마다 별칭(alias) 목록을 데이터로 갖고, 검색어를 정규화(소문자-공백 제거)해 별칭 전체와 대조. `pg_trgm`(문자 조각 유사도)으로 오타-띄어쓰기 교정.

```json
{ "name_ko": "지마켓 산스", "name_en": "Gmarket Sans",
  "aliases": ["지마켓산스","g마켓 산스","지마켓 폰트","gmarketsans","gmarket sans ttf","지마켓 서체"] }
```

- 정확 별칭 > 부분일치 > trgm 유사도 순 점수화. 상세 스키마-검색 함수는 v2.2 참조.
- 알리아스는 상세 페이지 본문에도 자연스럽게 포함(검색 노출 키워드).
- 검색은 브라우저가 아니라 Next.js 서버 라우트를 거쳐 Supabase에 묻는다(키 노출 방지).
- 결과 0건 검색어는 로그로 남겨 등록 우선순위에 활용(F-17).

### 6-1. 자동완성 제안 — 실시간 드롭다운 (F-19)

검색바에 글자를 입력하는 동안 매칭되는 폰트를 드롭다운으로 즉시 제안한다(눈누-카톡 검색과 동일한 UX 패턴). 결과 페이지 검색과 **같은 엔진-정규화-점수화**를 쓰되, 응답을 상위 N개(예: 8개)로만 제한해 가볍게 유지한다.

- **동작**: 입력 변화 → 디바운스(예: 150~200ms) → Next.js 서버 라우트가 별칭-초성-`pg_trgm` 매칭 상위 N개를 폰트명-제작사와 함께 반환. 브라우저가 아니라 서버 라우트를 거쳐 Supabase에 묻는다(키 노출 방지, 검색 F-04와 동일 경로).
- **매칭 범위**: 한글-영문-붙여쓰기-오타-초성(F-04-F-16 엔진 재사용). "ㅈㅁㅋ", "gmarketsans", "지마켓폰트" 모두 "지마켓 산스"를 제안.
- **UI**: 매칭 구간 하이라이트, 키보드 상하 탐색-엔터 이동, 항목 클릭 시 해당 폰트 상세로 직행. 슬롯 높이 사전 확보로 레이아웃 이동(CLS) 방지.
- **동적**: SSG와 무관하게 검색-클릭 집계처럼 동적으로 처리(12장 렌더링 정책과 일관).
- **빈약 방어**: 데이터 적은 초기엔 제안이 빈약할 수 있음 → 0건이면 드롭다운 대신 "결과 없음" 안내 + 실패어 로그(F-17)로 등록 우선순위 반영.

## 7. 인기 Top 10 트렌드 집계 설계 (신규)

차별점의 핵심이지만 이전 기획서에 설계가 없었다. 여기서 확정한다.

### 7-1. 무엇을 집계하나
우리는 파일을 호스팅하지 않으므로 "다운로드 수"가 아니라 **"우리 사이트에서 공식 페이지로 이동한 클릭 수"**를 집계한다. 표기도 반드시 **"이동 클릭 기준 인기"**로 — "다운로드 순위" 사칭 금지(정직성).

### 7-2. 개인정보 원칙
클릭 집계는 **익명 카운트만**. 폰트별로 "공식 링크가 눌렸다"는 이벤트만 기록하고, IP-개인 식별 정보-검색 문장은 저장하지 않는다(기획서 개인정보 원칙과 일관).

### 7-3. 데이터 구조 (v2.2 스키마에 추가)
```sql
-- 이동 클릭 이벤트 (익명, 개인식별 없음)
create table public.font_clicks (
  id         uuid primary key default gen_random_uuid(),
  font_id    uuid not null references public.fonts(id) on delete cascade,
  clicked_at timestamptz not null default now()
  -- IP-사용자 식별자 저장하지 않음
);
create index if not exists idx_clicks_font_time
  on public.font_clicks(font_id, clicked_at);

-- (트래픽 커지면) 일별 롤업 — raw 대신 집계로 조회
create table public.font_click_daily (
  font_id uuid not null references public.fonts(id) on delete cascade,
  day     date not null,
  count   int  not null default 0,
  primary key (font_id, day)
);
```

Top 10 쿼리(최근 7일 예시):
```sql
select f.*, sum(d.count) as weekly
from public.font_click_daily d
join public.fonts f on f.id = d.font_id
where d.day >= current_date - interval '7 days'
  and f.status = 'published'
group by f.id
order by weekly desc
limit 10;
```

### 7-4. 동작 흐름
- 사용자가 상세 페이지에서 "공식 페이지 이동" 클릭 → 서버 라우트가 `font_clicks`에 1건 기록 후 리다이렉트.
- 랭킹은 **일별 집계값**을 쓴다(실시간일 필요 없음). 새벽 cron이 raw → `font_click_daily` 롤업.
- 홈/트렌드 페이지는 SSG라 빌드/일별 재생성 시점의 랭킹을 보여준다. 카드의 "이동수"도 이 집계값.

### 7-5. 트레이드오프
- raw 이벤트를 매번 세면 트래픽 증가 시 부담 → 일별 롤업으로 완화. MVP는 raw + 기간 쿼리로 시작, 커지면 롤업 테이블 전환.
- 초기 클릭 데이터가 적으면 Top 10이 빈약 → 데이터가 쌓이기 전엔 "최신 등록" 등으로 대체하거나 기간을 넓힌다.
- 어뷰징(자기 폰트 클릭 조작) 가능 → 심해지면 동일 세션 중복 제한 등 방어(초기엔 과설계 안 함).

## 8. AdSense - 수익화

**⚠️  "adsense 감지 기능" = 애드블록(광고 차단) 감지로 해석. 다른 의미면 이 장 수정.**

- 승인: 공개 100페이지+ - 색인 시작 후 신청. 승인 후 루트에 **ads.txt**. 1차 거절 → 보강 후 재신청 1회, **2차 거절 → 무한 재신청 금지, 구조 재검토**(카카오 AdFit 등 임시 운영).
- 배치(광고 라이트 정책, 3장 KP0): 성장기(일 방문 500 미만)는 광고 0~1개만 — "눈누보다 가벼운 화면"이 차별점이므로 수익보다 체류-재방문 우선. 이후 상세 미리보기 아래 1 + 하단 1, 목록-컬렉션 인피드 1까지 단계 확대. 슬롯 높이 사전 확보(CLS 방지).
- 애드블록 감지: 초기엔 **정중한 안내 배너**(닫기 가능), 일 방문 500+ 안정 후 강제 방식 테스트. AdSense 콘솔의 "광고 차단 복구" 공식 기능부터 확인.
- 보조 수익(후속): 유료폰트 마켓 제휴(가입 조건 확인), 디자인 툴-강의 어필리에이트 — Tier C 페이지와 궁합.

## 9. 법적 안전선 — 절대 규칙

수익은 AdSense 계정 위에 있다. 저작권 사고는 수익 파이프를 끊는 사고다.

1. 유료폰트의 파일-다운로드 링크-"무료 다운" 문구 절대 금지. 정보와 공식 구매 링크만.
2. 무료폰트도 파일 직접 호스팅 금지. 링크만.
3. 링크는 **제작사 공식 도메인-공식 채널만**. 재업로드 링크 금지. 공식 페이지 사라지면 "배포종료" 상태로.
4. 미리보기: 웹폰트 임베딩 공식 허용분만 웹폰트, 나머지는 서버 생성 이미지. 변환-서브셋-자체 CDN 금지.
5. **타 아카이브(눈누 등) 문구 복제 금지.** 라이선스 요약은 제작사 공식 페이지(1차 출처)를 직접 읽고 작성.
6. "무료 대안" 표현: "비슷한 분위기-용도"까지만. 동일성 주장-유료폰트 폄하 금지.
7. 회사 로고-공식 사이트 스크린샷 무단 게재 금지.
8. "안전함/문제없음" 보증 표현 금지. 확인일-원문 링크라는 사실만. 하단 면책 고지 상시.
9. 신고 접수 → 48시간 내 보류 판단 → 재공개/폐기, 이력 기록.

## 11. 브랜딩 - 디자인 (요약, 상세는 브리프 v2)

- **브랜드**: FontAgit(폰트 아지트). 로고는 "FontAgit"처럼 A 대문자로 두 단어 경계를 보이고 한글 "폰트 아지트" 병기. 태그라인 "당신의 폰트 아지트".
- **FontAgent 구분**: 기존 소프트웨어 FontAgent와 철자 1글자 차이 → 한글 병기 + 로고 심볼 + "웹에서" 성격으로 구분.
- **층위 분리**(핵심): 콘텐츠 층(견본-목록-상세-라이선스)은 중립-조용(폰트가 주인공), 브랜드 층(로고-히어로-404-빈 상태-마이크로카피)은 아지트지기 인격. "차분한 바(bar)에 인격 있는 주인장".
- **색**: 무채색 베이스 + 딥 그린 포인트 `#2C5545`(기본값, 차별화). 안전 대안 딥 인디고. 다크모드 필수.
- **타이포**: UI는 Pretendard, 견본은 각 폰트. 분리 필수(견본 실패해도 정보 읽힘).
- **라이선스-근거는 인격 빼고 건조하게, 트렌드는 "이동 클릭 기준"으로.**
- 런칭 자산: 파비콘-OG 이미지(폰트 상세는 폰트명 동적 OG 이상적)-메타 태그.

## 12. 인프라 - 데이터 파이프라인 (요약, 상세는 v2.2)

3층 구조:
```
[로컬 머신]  →  [VM1 Supabase]  →  [VM2 Next.js]  →  [Cloudflare]  →  사용자
 무거운 배치      DB-스토리지-검색    SSG 서빙          캐시-속도
```
- **로컬 머신**: 크롤링-폰트 분석-이미지 생성 전담. 결과만 Supabase 업로드(서빙과 경합 없음).
- **VM1(오라클 4 OCPU/12GB)**: 셀프호스팅 Supabase(supabase.ollidam.com). DB + Storage(미리보기) + 검색 백엔드.
- **VM2(오라클 4 OCPU/12GB)**: Next.js **SSG**로 페이지 미리 생성 → Nginx 서빙. 검색-클릭 집계만 동적.
- **Cloudflare(무료 CDN)**: 정적 페이지-이미지 캐시로 부하-대역폭 절감.

렌더링: **SSG 우선**. 폰트 200개 규모면 빌드 몇 분이라 Node 서버(ISR) 불필요. 배치 후 또는 새벽 cron 재빌드. 수천 개로 커지면 ISR 전환.

파이프라인 자동/반자동: 구글폰트 수집-폰트 분석-이미지 생성-링크 점검은 **자동**, 알리아스-라이선스 요약-Tier B 발견은 **반자동(사람 검수-승인)**.

## 13. 데이터 모델 - 스키마

핵심 테이블(SQL 전문-인덱스-RLS-검색 함수는 v2.2 참조):
- `fonts` — 본체(이름-tier-라이선스 4상태-주의 조건-공식 URL-상태-확인일-기술 메타)
- `font_aliases` — 별칭 1:N(검색 핵심, trgm 인덱스)
- `font_alternatives` — 무료 대안(Tier C)
- `collections`, `collection_items` — 모음 페이지
- `review_queue` — 크롤링 후보 검수(운영 전용, RLS 잠금)
- `font_change_log` — 변경 이력(운영 전용, RLS 잠금)
- **`font_clicks`, `font_click_daily`** — 이동 클릭 집계(신규, 7장). 익명.

RLS: 공개 폰트-컬렉션만 anon 읽기, 운영 테이블은 잠금. 쓰기는 로컬 배치-검수 도구가 service_role로.
