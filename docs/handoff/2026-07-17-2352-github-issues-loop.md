## 무중단 루프 처리 프로토콜 (권장)
각 이슈마다 반복:
1. `gh issue view <n>` + `gh pr list --search "#<n>"`로 **이미 병합됐는지 먼저 확인** → 됐으면 스킵(수동 단계만)
2. 미완이면 브랜치 생성(develop 기준) → subagent-driven 구현
3. 테스트/빌드/lint 검증(로그 첨부)
4. PR 생성(**base=develop**) → 사용자에게 codex 리뷰는 직접 요청하도록 안내
5. #62 체크리스트 갱신 후 다음 이슈로
- 순서: MUST(#25,#28) → 에픽 #60(#56/#57/#55/#54) → #53 → #31 → 백로그(#2/#3/#8/#10)
- ⚠️ 매 이슈 착수 전 `git branch --show-current` 재확인(브랜치가 바뀌어 있을 수 있음)
